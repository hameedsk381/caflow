Calendar & Leaves — Which layout fits best?

Pick the approach for CAFlow's calendar module

A

Full Calendar View (Recommended)

Monthly/weekly grid showing all events — compliance deadlines, tasks, leave requests, hearings — colour-coded by type. Staff see their own events; admins see firm-wide view. Leaves submitted via a side panel.

Pros

  • Visual at-a-glance overview
  • Matches Turia's calendar exactly
  • Leaves integrated naturally

Cons

  • Needs a calendar library (react-big-calendar)
B

Timeline / List View

Chronological list of upcoming events grouped by day. Simpler to build, no calendar grid. Leaves shown as a separate tab on the same page.

Pros

  • Fast to build
  • Works well on mobile

Cons

  • Less visual, harder to spot gaps
  • Doesn't match Turia's look
C

Calendar + Leaves as Separate Pages

Full calendar grid on /calendar. Separate /leaves page with submit / approve workflow. Events shown on calendar but leave management is standalone.

Pros

  • Clean separation of concerns
  • Leaves page can grow independently

Cons

  • Two pages to navigate for one workflow
